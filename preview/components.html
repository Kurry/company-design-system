Best Practices for Cards & Surfaces

Do:

  • • Use Browser Company shadow tokens for consistent elevation
  • • Combine Tailwind utilities with custom design tokens
  • • Maintain consistent padding and spacing
  • • Use appropriate shadow depth for content hierarchy
  • • Include hover and focus states for interactive cards
  • • Use semantic HTML structure for accessibility

Don't:

  • • Override Browser Company shadow values unnecessarily
  • • Mix different shadow depths randomly
  • • Overcrowd cards with too much content
  • • Use excessive border radius that conflicts with brand
  • • Neglect responsive behavior on mobile devices
  • • Forget to provide clear action affordances

Line Heights

Tight Leading (leading-tight)

This paragraph demonstrates tight line height. Notice how the lines are closely spaced together, creating a compact appearance that works well for headings and short bursts of text where space is at a premium.

Normal Leading (leading-normal)

This paragraph uses normal line height, which is the default spacing. It provides good readability for most content types and is suitable for body text, maintaining a balance between compactness and breathability.

Relaxed Leading (leading-relaxed)

This paragraph demonstrates relaxed line height with extra spacing between lines. This creates a more open, airy feeling and can improve readability for longer form content or when you want to create a more luxurious reading experience.

Loose Leading (leading-loose)

This paragraph uses loose line height with generous spacing between lines. This creates maximum readability and is ideal for dense content, accessibility considerations, or when you want to create a very open, scannable layout.

Text Alignment & Decoration

Text Alignment

Left aligned text - Standard alignment for most Western languages and body content.

Center aligned text - Perfect for headings, quotes, and emphasized content.

Right aligned text - Useful for metadata, dates, and numeric content.

Justified text creates even margins on both sides by adjusting spacing between words. This can improve the appearance of dense text blocks but should be used carefully as it can create uneven word spacing.

Text Decoration

Normal text without any decoration

Underlined text for emphasis or links

Strikethrough text for deleted content

Overlined text for special emphasis

No underline - removes default link styling

Animation Best Practices

✅ Do

  • • Use transform and opacity for performance
  • • Apply will-change-transform for complex animations
  • • Use Browser Company duration tokens
  • • Leverage Tailwind's built-in animation classes
  • • Keep animations subtle and purposeful
  • • Test on slower devices
  • • Respect prefers-reduced-motion

❌ Don't

  • • Animate layout properties (width, height, etc.)
  • • Use animations longer than 500ms for micro-interactions
  • • Over-animate - less is more
  • • Ignore accessibility preferences
  • • Use complex animations on mobile
  • • Animate too many elements simultaneously
  • • Forget to test performance
